HTML semantic tag cheatsheet

By SupplePaws

The table below shows html tags and their semantic use

Tag Name Explanation
<article> article An article should make sense on its own, and it should be possible to distribute it independently from the rest of the web site.
<aside> aside The aside element defines some content aside from the content it is place in (like a sidebar).
<details> details The details tag specifies additional details that the user can open and close on demand.
<figcaption> figcaption The figcaption tag defines a caption for a figure element (below). The figcaption element can be placed as the first or as the last child of a figure element.
<figure> figure The figure tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.
<footer> footer The footer element defines a footer for a document or section.
<header> header The header element represents a container for introductory content or a set of navigational links.
<main> main The main tag specifies the main content of a document.
<mark> mark The mark tag defines text that should be marked or highlighted.
<nav> nav The nav element defines a set of navigational links.
<section> section The section element defines a section in a document.(typically with a heading.)
<summary> summary The summary tag defines a visible heading for the details element. The heading can be clicked to view/hide the details. The summary element should be the first child element of the details element.
<time> time The time tag defines a specified time or datetime.
